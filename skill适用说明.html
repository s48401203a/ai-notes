Skill（技能）是什么？怎么用？如何验证效果？

Skill 是一份可复用的“工作流说明书”，通常以 SKILL.md（含 YAML frontmatter）形式存在。 OpenCode 可在需要时通过内置 skill 工具加载 Skill 内容，让 Agent 按固定流程执行任务。

重要：Skill 不等于“工具”。Skill 更像一份约束与流程（Prompt/规范）；真正的执行仍依赖工具（bash/edit/webfetch/playwright 等）与权限配置。

概念与边界

概念本质典型用途和 Skill 的关系
Skill可复用的任务流程/规范（文档）把复杂任务“标准化”通过 skill 工具按需加载
Agent角色/权限/模型配置build / plan / 自定义 agentAgent 选择是否能加载/使用某些 skills
Tool可执行动作（读写/命令/浏览器等）bash/edit/webfetch/playwrightSkill 会约束工具的使用顺序与策略
Command（/命令）快捷触发一个固定 prompt 模板/test、/reviewCommand 里也可显式要求“使用某个 skill”
Rules（AGENTS.md/CLAUDE.md）项目级指令全局编码规范/安全约束Rule 是“长期背景”，Skill 是“按需加载的短流程”

OpenCode 中的 Skill

OpenCode 文档定义：Agent Skills 以 SKILL.md（含 YAML frontmatter 的 name/description）组织。 Agent 在需要时调用 skill 工具加载 Skill 内容，并按其中流程执行。

放置路径（发现机制）

位置路径说明
项目级.opencode/skills/<name>/SKILL.md推荐：随项目版本控制
全局~/.config/opencode/skills/<name>/SKILL.md个人/团队统一规范
Claude 兼容（项目）.claude/skills/<name>/SKILL.md可复用 Claude Code 生态的 skills
Claude 兼容（全局）~/.claude/skills/<name>/SKILL.md同上

命名规则（官方文档）

  • name 需要与目录名一致，且符合：^[a-z0-9]+(-[a-z0-9]+)*$
  • 长度：1-64；全小写；用单个 - 分隔；不能连续 --，也不能以 - 开头/结尾

权限与开关

可在 opencode.json 用 permission.skill 控制哪些 skill 允许加载（allow/ask/deny）；也可按 agent 覆盖或完全禁用 skill 工具。

{
  "$schema": "https://opencode.ai/config.json",
  "permission": {
    "skill": {
      "*": "allow",
      "internal-*": "deny",
      "experimental-*": "ask"
    }
  },
  "agent": {
    "plan": {
      "tools": { "skill": false }
    }
  }
}
