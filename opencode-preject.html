项目概览

OpenCode 是一个开源 AI Coding Agent，主打终端交互界面（TUI）与开发工具链集成。 README 给出的关键差异点包括：100% 开源、Provider-agnostic（不绑定模型提供商）、开箱即用的 LSP 支持、聚焦 TUI、以及客户端/服务器架构（可本机运行、远端驱动）。

适用场景：在本地代码库里用 TUI 进行探索、实现、修复、重构；或通过 GitHub Action 在 Issues/PR 里触发自动化任务。

安装与升级

README.zh-CN 提供了多种安装方式；官方建议 macOS/Linux 优先使用 tap（更新更快）。

一键安装（YOLO）

curl -fsSL https://opencode.ai/install | bash

包管理器（节选）

npm i -g opencode-ai@latest

scoop install opencode             # Windows
choco install opencode             # Windows
brew install anomalyco/tap/opencode # macOS/Linux 推荐
brew install opencode              # 官方 formula（更新频率较低）
paru -S opencode-bin               # Arch
mise use -g opencode               # Any OS
nix run nixpkgs#opencode           # 或 github:anomalyco/opencode 获取 dev

可从 Releases 或 opencode.ai/download 下载。

TUI Slash Commands（/命令）

在 TUI 输入 / + 命令名可快速执行动作；多数命令也有默认快捷键（Leader 默认为 Ctrl+X）。

命令 别名 作用 默认快捷键
/connect - 添加 Provider 并配置 API Key （以 UI 流程为主）
/compact /summarize 压缩当前会话（总结/减少上下文） Ctrl+X C
/details - 切换工具执行细节显示 Ctrl+X D
/editor - 用外部编辑器撰写消息（使用 EDITOR） Ctrl+X E
/exit /quit, /q 退出 OpenCode Ctrl+X Q
/export - 导出当前会话为 Markdown 并打开编辑器 Ctrl+X X
/help - 帮助/命令面板 Ctrl+X H
/init - 创建或更新 AGENTS.md（规则文件） Ctrl+X I
/models - 列出/选择可用模型 Ctrl+X M
/new /clear 新建会话 Ctrl+X N
/undo - 撤销上一条消息与相关文件修改（需要 Git） Ctrl+X U
/redo - 重做（仅在 /undo 后可用；需要 Git） Ctrl+X R
/sessions /resume, /continue 列出并切换会话 Ctrl+X L
/share - 分享当前会话（生成公开链接） Ctrl+X S
/unshare - 取消分享当前会话 （无默认快捷键）
/theme - 主题列表/切换 Ctrl+X T
/thinking - 显示/隐藏思考块（显示层面） （无默认快捷键）

来源：TUI 文档（opencode.ai/docs/tui/）

快捷键（Keybinds / Leader）

OpenCode 使用 leader 键机制减少与终端快捷键冲突。默认 leader 是 Ctrl+X。 例如：新建会话默认是 Ctrl+X 然后 N。

关键默认按键（高频）

动作 默认按键 说明
列出命令（命令面板） Ctrl+P 可快速查找命令/动作
切换 Agent（主 Agent 循环） Tab / Shift+Tab Build / Plan 等
切换模型 variant Ctrl+T 在 variants 之间循环（如 high/low 等）
新会话 Ctrl+X N 等价 /new
帮助 Ctrl+X H 帮助/命令面板
撤销/重做 Ctrl+X U / Ctrl+X R 依赖 Git（用于恢复文件改动）

自定义 Keybinds（opencode.json）

通过配置文件的 keybinds 字段自定义。可以用 none 禁用某个绑定。

{
  "$schema": "https://opencode.ai/config.json",
  "keybinds": {
    "leader": "ctrl+x",
    "session_new": "<leader>n",
    "session_compact": "<leader>c",
    "variant_cycle": "ctrl+t",
    "command_list": "ctrl+p",
    "scrollbar_toggle": "none"
  }
}

Agents（Build / Plan / General / Explore）

OpenCode 支持主 Agent（primary）与子 Agent（subagent）。 内置 primary：Build（默认，工具全开）、Plan（受限，只读/规划）；内置 subagent：General（多步任务/复杂搜索）、Explore（只读探索）。

  • 在会话中按 Tab 切换 primary agents（或配置 switch_agent）。
  • 在消息中使用 @general / @explore 调用子 agent。
  • 子会话导航：<leader>right/<leader>left 在 parent/child 之间循环。

来源：Agents 文档（opencode.ai/docs/agents/）+ README

技能（Skills）

OpenCode 支持通过 SKILL.md 定义可复用“工作流/规范”，并在需要时通过内置 skill 工具按需加载。 这类技能通常用来让 Agent 在执行复杂任务时遵循固定流程（例如：先侦察再行动、先出方案再执行、必须二次确认删除等）。

发现路径（官方文档）

  • 项目级：.opencode/skills/<name>/SKILL.md
  • 全局：~/.config/opencode/skills/<name>/SKILL.md
  • Claude 兼容：.claude/skills/<name>/SKILL.md、~/.claude/skills/<name>/SKILL.md

权限与开关

可通过 permission.skill 控制哪些技能可加载（allow/ask/deny）；也可针对某个 agent 禁用 skill 工具。

本机技能库（你已提供的目录）

  • Skills Catalog：../skills-catalog.zh-CN.html
  • 技能目录：D:\AI\opencode-Project\.opencode\skills
  • 本知识库总览：skill适用说明.html（含 2 个可复现实例）

来源：Agent Skills 文档（opencode.ai/docs/skills/）

CLI 命令（opencode ...）

CLI 不带参数默认启动 TUI，也可用子命令做自动化。

常见用法

# 启动 TUI
opencode

# 非交互模式：直接运行一次对话
opencode run "Explain how closures work in JavaScript"

主要子命令（按官方文档）

命令 用途 要点
opencode agent 管理 Agents create / list
opencode attach 连接已运行的 serve/web 后端 支持远端后端 + 本地 TUI
opencode auth 管理 Provider 凭据 登录信息存于 ~/.local/share/opencode/auth.json
opencode github 安装/运行 GitHub Agent install / run
opencode mcp 管理 MCP servers add / list / auth / debug
opencode models 列出模型 支持 --refresh
opencode serve 启动 headless HTTP 服务 可通过 OPENCODE_SERVER_PASSWORD 启用 Basic Auth
opencode web 启动带 Web UI 的服务 同样支持 Basic Auth
opencode session 管理会话 list 等
opencode stats 查看 token/cost 统计 支持按天数/模型/项目过滤
opencode export/import 导出/导入会话 导入支持 share URL
opencode upgrade 升级 支持指定版本/方法
opencode uninstall 卸载 支持 dry-run/keep-config/keep-data
opencode acp 启动 ACP server stdin/stdout nd-JSON

环境变量（节选）

变量 说明
OPENCODE_CONFIG 指定配置文件路径
OPENCODE_CONFIG_DIR 指定配置目录（agents/commands/plugins 等）
OPENCODE_CONFIG_CONTENT 内联 JSON 配置内容
OPENCODE_SERVER_PASSWORD 为 serve/web 启用 Basic Auth（用户名默认 opencode，可用 OPENCODE_SERVER_USERNAME 覆盖）
OPENCODE_DISABLE_LSP_DOWNLOAD 禁用自动下载 LSP server
OPENCODE_DISABLE_CLAUDE_CODE* 禁用读取 .claude（prompt/skills 等兼容层）
OPENCODE_EXPERIMENTAL 启用实验特性（含 LSP tool 等）

配置（opencode.json / JSONC）

OpenCode 支持 JSON 与 JSONC（带注释）。配置会按优先级合并，而不是整体替换。

配置来源优先级（后者覆盖前者的冲突键）

  1. Remote config（`.well-known/opencode`）
  2. Global config：~/.config/opencode/opencode.json
  3. Custom config：OPENCODE_CONFIG
  4. Project config：项目根目录 opencode.json
  5. .opencode 目录（agents/commands/plugins/skills/tools/themes 等）
  6. Inline：OPENCODE_CONFIG_CONTENT

示例（节选）

{
  "$schema": "https://opencode.ai/config.json",
  "theme": "opencode",
  "model": "anthropic/claude-sonnet-4-20250514",
  "autoupdate": true,
  "share": "manual",
  "permission": {
    "bash": { "*": "ask", "git *": "allow", "rm *": "deny" },
    "edit": "ask"
  }
}

来源：Config 文档（opencode.ai/docs/config/）

权限（Permissions）

通过 permissionallow/ask/deny。

支持对象语法对不同输入做细粒度控制（例如限制危险 bash 命令、只允许编辑指定目录）。

{
  "$schema": "https://opencode.ai/config.json",
  "permission": {
    "bash": {
      "*": "ask",
      "git *": "allow",
      "grep *": "allow",
      "rm *": "deny"
    },
    "edit": {
      "*": "deny",
      "packages/web/src/content/docs/*.mdx": "allow"
    }
  }
}

来源：Permissions 文档（opencode.ai/docs/permissions/）

MCP Servers（外部工具扩展）

MCP（Model Context Protocol）用于把外部工具接入 OpenCode。支持 Local 与 Remote，并支持 OAuth（自动处理）。 注意：启用 MCP 会增加上下文 token 消耗，建议按需启用。

常见操作

opencode mcp add
opencode mcp list
opencode mcp auth <server-name>
opencode mcp logout <server-name>
opencode mcp debug <server-name>

来源：MCP servers 文档（opencode.ai/docs/mcp-servers/）

LSP 支持

OpenCode 可集成 Language Server Protocol（LSP）用于诊断与代码理解。 文档列出了内置/可自动安装的 LSP server（TypeScript、ESLint、Pyright、gopls 等）。

禁用自动下载：可设置 OPENCODE_DISABLE_LSP_DOWNLOAD=true。 
{
  "$schema": "https://opencode.ai/config.json",
  "lsp": {
    "typescript": { "disabled": true }
  }
}

来源：LSP 文档（opencode.ai/docs/lsp/）

GitHub 集成（/opencode /oc）

OpenCode 可在 GitHub Issues/PR 评论中通过提及 /opencode 或 /oc 触发执行，运行在 GitHub Actions runner 内。

安装

opencode github install

支持的触发事件（官方文档列出）

事件 触发方式 用途
issue_comment issue/PR 评论包含 /oc 或 /opencode 解释/修复/实现，可能自动开 PR
pull_request_review_comment PR 行内评论包含 /oc//opencode 针对具体文件/行执行修改与建议
issues issue opened/edited（需要 prompt） 自动化分诊/回应
pull_request PR opened/synchronize/reopened（可默认 review） 自动 review
schedule cron（需要 prompt） 定时任务
workflow_dispatch 手动触发（需要 prompt） 按需执行

手动 workflow 示例（节选）

name: opencode
on:
  issue_comment:
    types: [created]
  pull_request_review_comment:
    types: [created]
jobs:
  opencode:
    if: |
      contains(github.event.comment.body, '/oc') ||
      contains(github.event.comment.body, '/opencode')
    runs-on: ubuntu-latest
    permissions:
      id-token: write
    steps:
      - name: Checkout repository
        uses: actions/checkout@v6
        with:
          fetch-depth: 1
          persist-credentials: false
      - name: Run OpenCode
        uses: anomalyco/opencode/github@latest
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
        with:
          model: anthropic/claude-sonnet-4-20250514

来源：GitHub 文档（opencode.ai/docs/github/）

自定义命令（Commands）

OpenCode 支持自定义 TUI 命令（以 /my-command 形式）。可通过配置或在 .opencode/commands/ 下创建 markdown 文件。 这类命令是对内置命令（/init、/undo、/redo、/share 等）的补充。

用 markdown 定义命令（示例）

---
description: Run tests with coverage
agent: build
model: anthropic/claude-3-5-sonnet-20241022
---
Run the full test suite with coverage report and show any failures.
Focus on the failing tests and suggest fixes.

占位符

  • $ARGUMENTS / $1、$2：命令参数
  • !`command`：把 bash 输出注入 prompt
  • @file：引用文件内容

来源：Commands 文档（opencode.ai/docs/commands/）

FAQ / 最佳实践

如何选择模型？

文档建议用 /models 选择模型；并在 Models 文档里列出了一些推荐模型（示例：GPT 5.2、Claude Sonnet/Opus 等）。

建议：在 TUI 里用 Ctrl+T 切换 variants，配合任务复杂度选择 reasoning effort。

如何避免误操作？

通过 permission 让危险操作变为 ask/deny（尤其是 bash 的 rm、git push 等）。

会话分享注意事项

分享会话会生成公开链接（任何拥有链接的人都可访问）；敏感项目建议在 config 中禁用 share。

{ "$schema": "https://opencode.ai/config.json", "share": "disabled" }
