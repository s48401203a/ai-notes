Windows密码修改工具使用说明

脚本说明

本工具提供了两个版本的Windows密码修改脚本：

  • ChangePassword.ps1 - PowerShell版本
  • ChangePassword.bat - 批处理文件版本

使用方法

方法一：直接双击运行（推荐）

  1. 找到脚本文件（ChangePassword.bat 或 ChangePassword.ps1）
  2. 直接双击运行
  3. 按照提示输入新密码并确认
  4. 等待脚本执行完成

方法二：使用命令提示符运行

  1. 按下 Win + R 键，输入 cmd 并按Enter
  2. 切换到脚本所在目录：cd /d d:\AI\知识库
  3. 运行脚本：ChangePassword.bat

注意事项

  • 权限要求：修改密码需要管理员权限，建议右键点击脚本，选择"以管理员身份运行"
  • 密码要求：密码不能为空，建议长度至少6个字符，应包含字母、数字或特殊字符
  • 使用安全：请确保在安全的环境中运行脚本，不要与他人共享您的密码

脚本特点

  • 中文友好界面
  • 密码确认机制
  • 密码长度检查
  • 详细的错误提示
  • 支持各种Windows系统
  • 隐藏密码输入（PowerShell版本）

系统要求

  • Windows 7 及以上版本
  • 支持PowerShell 2.0+（用于密码输入隐藏功能）

CLAUDE.md 文件说明

此文件为 Claude Code (claude.ai/code) 提供项目上下文指导。

项目概述

这是一个技术知识库项目，主要存储HTML格式的技术文档和参考资料。内容聚焦于Claude Code使用指南、错误代码查询、认证问题解决方案等实用文档。所有文档均为独立的HTML页面，包含嵌入式样式。

常用开发命令

本地预览

# 启动本地HTTP服务器预览HTML文档
python3 -m http.server 8000
# 然后在浏览器访问 http://localhost:8000

代码格式化

# 格式化HTML文件(需要先安装Prettier)
npx prettier --write "*.html"

# 检查HTML结构问题
tidy -qe *.html

内容组织规范

  • 文件命名：使用描述性的、支持中英双语的文件名，格式为 <主题>-<用途说明>.html
  • 文档位置：HTML知识文档直接放在根目录
  • 共享资源：图片、CSS等共享资源应放在 assets/ 目录下

编码规范

  • 使用4个空格缩进
  • 内联样式集中在文档头部的单个 <style> 块中
  • 使用小写、连字符分隔的class和id命名（如 .error-table）
  • 优先使用语义化标签（<section>, <article>, <nav>）

测试流程

  1. 本地预览测试：修改后使用 python3 -m http.server 8000 启动服务并在浏览器中预览
  2. 跨浏览器测试：在Chrome和Firefox（或Edge）中验证页面布局一致性
  3. 链接验证：手动检查所有链接和锚点目标是否可正确解析

安全注意事项

  • 不要在HTML中嵌入API密钥或专有代码片段
  • 嵌入日志前需清理敏感信息
  • 如创建配置文件（如 .env），应添加到 .gitignore 中

Repository Guidelines

Project Structure & Module Organization

  • HTML knowledge articles live at repository root (e.g., claude-code-errors错误代码查询.html)
  • Shared styling is embedded per page; when adding common assets (images, CSS), place them under assets/
  • Keep filenames descriptive and bilingual when appropriate: <topic>-<usage说明>.html

Build, Test, and Development Commands

# serve the repository locally for quick previews
python3 -m http.server 8000

# format new or edited HTML before committing
npx prettier --write "*.html"

# surface structural HTML issues
tidy -qe *.html

Coding Style & Naming Conventions

  • Indent HTML and CSS with four spaces
  • Keep inline styles grouped under a single <style> block near the document head
  • Use lowercase, hyphenated class and id names (e.g., .error-table)
  • Favor semantic tags (<section>, <article>, <nav>) over excessive <div> usage

Testing Guidelines

  • After changes, open the served page in Chrome and Firefox (or Edge) and confirm layout parity
  • Validate links and anchor targets manually
  • When adding scripts, include console logging guards and verify there are no errors in DevTools

Commit & Pull Request Guidelines

  • Use Conventional Commits (e.g., docs: add print spooler diagnostic guide)
  • Write commit bodies describing rationale, sources, and testing steps
  • Pull requests must include: summary of affected pages, before/after screenshots for visual edits

Security & Configuration Tips

  • Exclude API keys or proprietary snippets; scrub logs before embedding them in HTML
  • Keep local .env or credentials files out of the repository
