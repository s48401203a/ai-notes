Cherry Studio

基于 Electron 的全能跨平台 AI 桌面客户端

技术架构: Electron 上手难度: ⭐⭐ 是否需要代理: 是(云端) / 否(本地) 适用场景: 聚合聊天 / 知识库 / 翻译

🛠️ 技术架构与环境

  • 核心框架: Electron (Chromium + Node.js)，确保了 Windows, macOS, Linux 三端一致的体验。
  • 本地数据: 采用 SQLite/LocalForage 存储对话记录，隐私数据（API Key）加密存储于本地，不上传云端。
  • 扩展机制: 支持 MCP (Model Context Protocol) 协议连接外部服务，支持插件系统（如 Web 搜索）。
  • 环境要求:
    • Windows 10/11 (x64)
    • macOS 12+ (Apple Silicon/Intel)
    • 网络: 访问 OpenAI/DeepSeek API 需确保网络连通性（或配置代理）。

📖 核心功能使用教程

1. 接入 DeepSeek R1/V3 (硅基流动版)

为什么用硅基流动？ DeepSeek 官方 API 常不稳定，硅基流动 (SiliconFlow) 提供更稳定的企业级服务，且 Cherry Studio 内置了专有配置项。
  1. 注册 SiliconCloud 获取 API Key。
  2. 打开 Cherry Studio 设置 -> 模型服务 -> 选择 SiliconFlow。
  3. 粘贴 API Key -> 点击“检查” -> 开启 deepseek-ai/DeepSeek-R1 模型开关。
  4. 回到对话页，顶部模型选择器选中 R1，即可开始“深度思考”。

2. 搭建本地 RAG 知识库

  1. 点击左侧边栏的“知识库”图标 -> 新建知识库。
  2. 拖入 PDF / Word / Markdown 文件（支持批量）。
  3. 系统会自动进行切片和向量化（Embedding）。
  4. 在对话框下方的“关联知识库”按钮中勾选该库，AI 回答时将自动检索文档内容。

💡 可实践案例 (Use Cases)

案例一：沉浸式翻译阅读

场景: 阅读英文论文或技术文档。
操作: 将 PDF 拖入知识库。使用 Gemini 1.5 Pro (长窗口模型)。
指令: "请总结这篇论文的核心创新点，并用中文解释第三章的技术细节。"

案例二：多模型交叉验证 (MoE)

场景: 编写复杂的正则代码。
操作: 开启“多模型并发”模式（需 v1.6+）。
配置: 同时勾选 Claude 3.5 Sonnet 和 DeepSeek V3。
结果: 发送同一个 Prompts，左右分屏对比两者的代码实现，取长补短。

⚠️ 注意事项

  • 代理配置: 若无法检查更新或连接 OpenAI，请在设置 -> 网络 -> 代理中填入 http://127.0.0.1:7890 (端口视你的软件而定)。
  • Ollama 连接: 若使用 Docker 版 Ollama，需配置环境变量 OLLAMA_HOST=0.0.0.0 允许外部连接。
