1. 项目概述

Password Policy Manager 是一款专为企业环境设计的 Windows 密码策略管理工具，通过火绒安全软件进行静默分发部署。该工具的核心功能是：

一次性密码提醒

部署后持续提醒用户修改密码，直到用户完成密码更改

自动卸载机制

检测到密码修改后，自动清理所有部署文件和计划任务

静默部署

支持火绒集中分发，SYSTEM权限静默执行，用户无感知

智能账户识别

自动跳过Microsoft账户，仅针对本地账户生效

适用场景

  • 企业内部强制密码策略执行
  • 新员工入职密码修改提醒
  • 安全合规审计要求
  • 批量终端密码策略部署

技术栈

组件 技术 说明
部署脚本 Windows Batch (.bat) 主部署逻辑，兼容性最佳
监控脚本 PowerShell (.ps1) 密码检测、通知显示
清理脚本 Windows Batch (.bat) 避免安全软件拦截
通知系统 Windows Toast API 原生系统通知
任务调度 Windows Task Scheduler 登录触发 + 定时触发

5. 工作流程

部署流程

1 火绒分发执行 - SYSTEM 权限运行 BAT 脚本
↓
2 密码策略配置 - 使用 secedit 设置密码长度(8位)、复杂度、历史(3次)
↓
3 解码并创建文件 - Base64 解码生成 PS1、VBS、BAT 文件
↓
4 创建计划任务 - 注册 Logon、Daily、Cleanup 三个任务
↓
5 自删除脚本 - 如有 /autodel 参数，删除分发脚本自身

监控流程

1 触发执行 - 用户登录(延迟30秒) 或 每日9:00
↓
2 账户检测 - 检查是否为 Microsoft 账户，是则跳过并卸载
↓
3 通知权限 - 启用系统通知、禁用勿扰模式
↓
4 密码比对 - 对比当前密码日期与部署时记录
↓
5 结果处理 - 未修改→显示通知 | 已修改→触发卸载

卸载流程

1 创建标记 - 监控脚本检测到密码变更，创建 cleanup.flag
↓
2 等待清理 - Cleanup 任务每5分钟检查一次标记文件
↓
3 删除任务 - 删除 Logon、Daily、Cleanup 三个计划任务
↓
4 删除目录 - 递归删除 C:\ProgramData\PasswordPolicyManager
完整时间线：用户修改密码 → 下次登录后30秒检测 → 创建清理标记 → 最多5分钟后完全删除

6. 使用方法

火绒分发部署 (推荐)

配置项 值
分发文件 PasswordPolicy_OneTime_v1.4.8.bat
分发方式 以系统权限运行
存储位置 系统盘根目录 (C:\)
运行参数 /autodel
重要：运行参数填写 /autodel 可在部署完成后自动删除分发脚本，保持系统整洁。

手动部署

:: 方式1：右键以管理员身份运行 (脚本保留) 右键 PasswordPolicy_OneTime_v1.4.8.bat → 以管理员身份运行 :: 方式2：命令行部署 (脚本自删除) cmd /c "C:\PasswordPolicy_OneTime_v1.4.8.bat" /autodel :: 方式3：PowerShell 部署 Start-Process -FilePath "C:\PasswordPolicy_OneTime_v1.4.8.bat" -ArgumentList "/autodel" -Verb RunAs

验证部署

:: 检查部署文件 dir "%ProgramData%\PasswordPolicyManager" :: 检查计划任务 schtasks /Query /TN "PasswordMonitor_Logon" schtasks /Query /TN "PasswordMonitor_Daily" schtasks /Query /TN "PasswordMonitor_Cleanup" :: 查看部署日志 type "%ProgramData%\PasswordPolicyManager\deploy.log" :: 查看当前状态 type "%ProgramData%\PasswordPolicyManager\status.txt"

手动卸载

:: 删除计划任务 schtasks /Delete /TN "PasswordMonitor_Logon" /F schtasks /Delete /TN "PasswordMonitor_Daily" /F schtasks /Delete /TN "PasswordMonitor_Cleanup" /F :: 删除部署目录 rd /S /Q "%ProgramData%\PasswordPolicyManager"

7. 技术细节

密码变更检测算法

采用三重检测机制确保准确识别密码变更：

# 方法1：密码日期比对 $currentDate = (Get-Date).AddSeconds(-$userObj.PasswordAge.Value).ToString("yyyy-MM-dd") if ($currentDate -gt $recordedDate) { $passwordChanged = $true } # 方法2：PasswordAge 增量检测 $expectedAge = $recordedAge + ((Get-Date) - $deployTimestamp).TotalSeconds if ($currentAge -lt ($expectedAge - 60)) { $passwordChanged = $true } # 方法3：PIN账户特殊处理 if (-not $wasReliable -and $isReliable -and $currentAge -lt 86400) { $passwordChanged = $true }

Toast 通知实现

# 使用 Windows Runtime API [Windows.UI.Notifications.ToastNotificationManager, Windows.UI.Notifications, ContentType = WindowsRuntime] # Toast XML 模板 (scenario="reminder" 确保持久显示) <toast scenario="reminder" duration="long"> <visual> <binding template="ToastGeneric"> <text>电脑密码修改提醒</text> <text>请您尽快修改电脑密码为强密码...</text> </binding> </visual> <actions> <action content="知道了" arguments="dismiss" activationType="system"/> <action content="稀后提醒" arguments="snooze" activationType="system"/> </actions> </toast>

9. 常见问题

Q: 通知没有弹出怎么办？

A: 脚本已自动启用通知权限。如仍无效，请手动检查：

  • 设置 → 系统 → 通知 → 确保"通知"已开启
  • 检查"专注助手/勿扰模式"是否关闭
  • 确认 PasswordPolicyManager 应用的通知权限

Q: 修改密码后多久会自动删除？

A: 最多 5-6 分钟。流程：登录后30秒检测 → 创建清理标记 → 最多5分钟后清理任务执行删除。

Q: 火绒弹出拦截提示怎么办？

A: v1.4.7+ 版本已解决此问题。如使用旧版本，请升级到 v1.4.8。

Q: 如何验证部署成功？

:: 检查文件 if exist "%ProgramData%\PasswordPolicyManager\PasswordMonitor.ps1" echo 部署成功 :: 检查任务 schtasks /Query /TN "PasswordMonitor_Logon" && echo 任务已创建

Q: 支持哪些 Windows 版本？

A: Windows 10 / Windows 11，包括企业版、专业版、家庭版。

Q: Microsoft 账户用户会收到提醒吗？

A: 不会。脚本自动检测并跳过 Microsoft 账户，仅对本地账户生效。
