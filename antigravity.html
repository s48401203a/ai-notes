Google Antigravity

Agent-First IDE: 重塑人机协作的代理式开发环境

技术架构: Client-Server (MCP) 上手难度: ⭐⭐⭐⭐ (高) 是否需要代理: 是 (Gemini) 核心理念: 自主代理 (Autonomous Agents)

🛠️ 技术架构 (Agent-First Architecture)

不同于 Copilot 的"补全"模式，Antigravity 采用代理优先架构：

  • 双视图设计:
    • Editor View: 传统代码编辑器，用于人类编写。
    • Manager View: 代理工作区，AI 在此拆解任务、运行终端命令、浏览网页。
  • MCP 协议集成: 通过 Model Context Protocol 连接外部工具（如 PostgreSQL, Linear, GitHub）。AI 不仅能写代码，还能直接操作数据库或管理工单。
  • Artifacts 系统: AI 生成的中间件（技术方案 implementation_plan.md, 任务列表 task.md）均以文件形式存在，确保可追溯、可版本控制。

⚡ 核心功能与环境要求

  • Browser Subagent: 内置无头 Chrome，AI 可自主查阅文档、抓取网页内容甚至测试 Web UI。
  • Terminal Control: AI 拥有 Shell 访问权限，可执行 npm install, git commit, pytest 等操作。
  • 环境要求:
    • OS: macOS / Windows (WSL2 推荐) / Linux。
    • 网络: 必须能访问 Google API (Gemini)。中国大陆用户必须配置 VPN/代理。
    • 依赖: 需安装 Chrome 浏览器以供 Subagent 使用。

⌨️ 常用指令与快捷键

功能 快捷键 (Mac/Win) 说明
Toggle Agent (唤出代理) ⌘/Ctrl + E 在 Editor 和 Manager 视图间切换。
Inline Command (内联指令) ⌘/Ctrl + I 在光标处直接下达修改指令（Diff模式）。
Toggle Terminal Ctrl + ` 显示/隐藏终端面板。
Accept Suggestion Tab 接受 AI 的灰色预测代码。

📖 使用教程: 标准工作流 (SOP)

Step 1: 任务定义 (Prompting)

不要只说"修复Bug"，要像给初级工程师写 Ticket 一样描述：

"请把登录页面的背景色改为深蓝色 #0f172a，并添加 Google OAuth 登录按钮。参考官方文档实现。"

Step 2: 规划与审核 (Planning)

AI 会生成 task.md 和 implementation_plan.md。务必阅读这些文档！

  • 检查它是否理解了你的需求。
  • 检查它是否准备修改正确的文件。
  • 点击 "Approve" 继续，或 "Reject" 并提供反馈。

Step 3: 执行与干预 (Execution)

AI 开始写代码。你可以看着它操作终端和编辑器。如果发现它走偏了，随时按 Ctrl+C 中断并给出新的指示。

Step 4: 验证 (Verification)

任务结束后，查看 walkthrough.md。AI 会贴出它的测试截图或运行结果。确认无误后，让它提交 Git Commit。

💡 实战案例 (Practical Cases)

案例一：老旧项目重构

挑战: 接手一个没有文档的 Python 遗留项目。
操作: 让 Antigravity 扫描全项目 (list_dir, view_file)，生成一份 README.md 和架构图。它会自动分析依赖关系并理清逻辑。

案例二：自动化测试编写

挑战: 为 React 组件补全单元测试。
操作: "为 components 目录下的所有 .tsx 文件编写 Jest 测试用例，要求覆盖率 80% 以上。" AI 会自动运行测试，看报错，修代码，直到测试通过。
