AI 学习知识库技术文档

版本：3.0

最近更新：2026-02-02

维护人员：作者

项目类型：离线可用的单文件 HTML 知识库

1. 项目概述

项目名称：技术知识库（Tech Knowledge Base）

创建时间：2025 年

定位：聚合 AI 学习、工具说明、运维实践与项目技术文档的离线知识库。

设计方向：Notion 布局 + Linear 配色，强调清晰层级、可读性与可维护性。

2. 项目目标

  • 收敛知识入口：把分散的 AI 工具、运维经验与项目实践统一收敛到一个离线入口。
  • 可复用输出：沉淀可复用的教程、提示词、流程与技术说明文档。
  • 离线可用：所有文档为单文件 HTML，可在无网络环境直接浏览。
  • 低维护成本：不依赖构建工具或框架，修改即生效。

3. 目标用户

  • 具备基础工程背景，想系统掌握 AI 协作工具与流程的用户。
  • 需要可复制实践模板的运维、产品、研发与技术管理人员。
  • 需要面向分享/教学的项目案例与文档素材的团队。

4. 项目结构

知识库/
├── *.html # HTML 格式知识文档
├── index.html # 首页入口
├── doc-center.html # 文档中心
├── AGENTS.md # 代码库规范（如存在）
├── CLAUDE.md # Claude Code 项目说明（如存在）
├── todo.md # 任务追踪（可选）
└── assets/ # 共享资源目录

5. 设计架构

5.1 布局风格

  • 双栏布局：左侧导航 + 中间内容区，便于快速定位章节。
  • 顶部导航：站点名称、返回入口与主题切换。
  • 响应式：移动端隐藏侧栏，仅保留正文阅读区。

5.2 视觉方向

  • 配色：Linear 深色体系 + 亮色对照主题。
  • 组件形态：卡片化段落承载内容，边界与间距清晰。
  • 内容密度：以可读性优先，避免重度装饰。

6. 核心功能

功能 说明 实现方式
离线可用 所有文档均为单文件 HTML，无后端依赖 原生 HTML + 内联 CSS/JS
主题切换 深色/浅色主题可切换 CSS 变量 + localStorage
文档中心检索 基于标题/描述/标签过滤 前端实时筛选
章节导航 滚动时高亮当前章节 滚动监听 + anchor 匹配
移动端适配 在手机/平板正常阅读 媒体查询

7. 文档中心页面结构

doc-center.html 是整个知识库的入口，结构与内容分类如下：

  • AI 编程助手：国外服务（需翻墙）与国内工具（直连）两类分区。
  • AI 学习与提示工程：工具对比、提示词工程、学习经验、项目实战。
  • 项目文档与技术规范：知识库技术文档与其他项目技术说明。
  • IT 运维与系统工具：运维实践、系统故障排查与工具说明。
  • 知识库项目文档：项目说明、规范、进展与协作指南。
  • 归档文档：旧版与合并内容的归档记录。

页面包含 Hero 区、教学导览区、卡片化文档列表与归档列表，支持搜索与折叠。

8. CSS 设计系统

  • 主题体系：使用 :root 定义变量，[data-theme="light"] 覆盖。
  • 颜色语义：背景、文本、边框、强调色分离，便于整体换肤。
  • 组件尺寸：统一圆角与边距，避免视觉不一致。
  • 命名规范：小写、连字符命名（如 .nav-link）。
:root {
  --bg: #0b1020;
  --panel: #121a2b;
  --accent: #6aa6ff;
}

[data-theme="light"] {
  --bg: #ffffff;
  --panel: #ffffff;
  --accent: #2b6bff;
}

9. JavaScript 模块

核心页面的 JS 主要用于交互增强，保持轻量、无框架依赖：

  • 主题切换：使用 localStorage 保存 ai-kb-theme。
  • 导航高亮：滚动时匹配当前 section 并高亮。
  • 文档中心搜索：标题/描述/标签/关键词过滤。
  • 折叠状态：保存 ai-kb-collapsed 到本地。
  • 统一工具栏：部分页面提供字体大小与搜索跳转能力。

10. 维护指南

10.1 更新现有文档

  1. 定位目标 HTML 文件并编辑对应 <section> 内容。
  2. 保持语义标签与层级一致，避免破坏布局。
  3. 更新 <title> 与 <meta description>。

10.2 新增文档

  1. 复制现有模板文件作为起点。
  2. 更新标题、描述、正文与导航。
  3. 在 doc-center.html 中增加文档卡片与关键词。

10.3 本地预览

python3 -m http.server 8000
# 浏览器打开 http://localhost:8000

修改后建议检查：页面布局、目录锚点、链接可用性与移动端阅读体验。
